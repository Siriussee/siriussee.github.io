进来这里也整整一年了，各位大佬的软件安利了吃了不少。现如今，看到各位新晋预备役软件工程师们还在用着令人感到难堪的软件，我便萌生了这样一个想法——把我当年吃过的安利再安利给你们。希望这些软件的存在能使各位的生活变得更加便利把。

# Bandzip

[Bandzip](xx)是BANDSOFT旗下一款免费的压缩软件。它最大的优点就是美观，和没有广告。

## 外观

这种情况下，只需要一张图就能说明问题：

![pic-of-bandzip](xx)

好看。跟7zip比起来简直就是一个天一个地。

此外，Bandzip对于分辨率的适配是极好的。从前在1920*1080的分辨率下用着快压，感觉问题不大，就是偶尔要一个权限来更新而已。更新了装备之后再看快压，在更高的分辨率下已经糊成了像素点，基本上是不能用了。相对的，Bandzip无论在哪个分辨率下都表现优秀，值得称道。

## 没有广告，孑然一身

360压缩，好压快压，Bandzip，这几个软件，我也各自都用过小半年了，还是Bandzip最清爽。

360压缩，不用谈，带这个数字的软件必然有一个在你家电脑上开枝散叶的梦想。所以，我个人的建议是，删光，然后不信任其证书，最后组策略以蔽之，永绝后患。

好压，全程2345好压，标榜“永久免费无广告”。事实似乎也的确如此。在使用好压的这段日子里，2345旗下的软件一个也没有住进来。这一点也证明了好压官网上的大字并非虚假宣传。只是有一点我比较在意，好压偶尔会向你要管理员权限来升级，这个弹窗令人烦躁。

Bandzip，是真的没有广告，而且，当它需要升级的时候，从来都只在你主动打开它时才提醒你，不会在我忙的不行的时候推送一个微小的更新。起码在我需要它的时候，它总是很安静，不闹腾。

## 性能

关于性能方面，我也不是什么专业人士，也不好意思下一个主观论断，就把这一块先放着，留待日后有空时再做一个时间对比。

# Snipaste

[Snipaste](xxx)是一款免费的截图软件，支持32位和64位的Windows 8 和 Windows 10 系统，使用简体中文。

在使用之前，我建议你将它加入启动项，这样就不必每次要用的时候再打开它了。

## 本职，截图（不）

Snipaste的默认截图快捷键是F1。按下F1，屏幕上就会出现截图的选择范围。选择好范围之后单击左键确定，就可以在截图范围左下角的工具栏中选择相应的工具对截图进行处理了。通常来说，我最常用的工具是添加文本，添加箭头和模糊工具。另外，在这个边栏里面还可以选择将截图保存到桌面。

如果各位熟悉QQ截图的话，拿到这个软件就可以立刻开始使用了，因为它的交互其实与QQ截图并没有多大的差距。

## 还有一件事，贴图

既然是这样那我为什么不选择QQ截图？反正我一年到头都挂着QQ的。那是因为Snipaste还有一个贴图功能。

什么叫做贴图？顾名思义，就是把图片贴在屏幕上。选择一个图片文件并复制，再按下贴图快捷键（我的是F3），这幅图片就会浮动到你的屏幕最前端。

贴图有什么用呢？它能够精简对话框，释放屏幕空间。

试想一个场景：我需要依照某个教程的指导进行算法设计。如果我把教程的网页和代码编辑器各放一半，这样两边大小都不上不下；若要是把大小调节一下，教程小了看起来不舒服，代码框小了，代码打起来也不舒服。此时如果把教程的重要部分截图，再贴图放进最大化的代码编辑器界面，万事大吉。

如果原尺寸的贴图让你感到不适，你甚至可以自由调节它的缩放比和不透明度。

## 还能简单的写写画画

在截图/贴图的界面，作者提供了编辑功能：

- 矩形/椭圆
- 直线/半透明直线（记号笔）
- 马赛克
- 文字

这些功能在处理数电实验数据的时候为我提供了极大的便利。

要处理一副示波器的照片，为它增加信号源、毛刺分析和数字表示，要是以前，那就肯定离不开 Photoshop 之类的专业图像处理应用了。现在有了 Snipaste，我可以轻松地在在贴图界面划线并标注。从此再也不用打开臃肿缓慢的 photoshop。

# Typora

Typora 是一个简洁明快的 Markdown 编辑器，它非常适合 MD 入门玩家。看上它的原因，是因为它所见即所得。

## 所见即所得

当我初学 markdown 语法的时候，我对那一长串意义不明的语法是很抗拒的。这些符号对我来说就是一大堆影响阅读的垃圾，只能看渲染出来的界面编辑文档。所以，我看到 Typora 的第一眼就对上了：这东西不需要我看源码！

单击左下角的`</>`标记，我们也可以直接看源码。

## easy copy

Typora 可以简单地把从 HTML 上复制下来地格式应用为 MD 格式。这样就省去了我重写一遍格式的痛苦。

## export

Typora 提供多种输出格式。

- HTML
- PDF
- docx（需要安装某些插件）
- LaTeX
- 等等

它渲染出来的 PDF 尤其值得称道，我的多篇实验报告/项目报告均出自它手。

## 然而，某些时候它会抽风

在所见即所得的模式下，Typora 不一定会如你所愿。这种时候我不得不选择进入源码界面进行编辑。

然而此时又出现一个问题：即便在源码模式下，所见即所得模式中的自动补全逻辑依然会生效。也就是说，所见即所得模式下的问题并不能得到解决。即便强行把源码改成正确的模式了。当你回到所见即所得模式下，所有已修正的错误都会复原。

> XD

现在我转投 VScode 来写我的 MD 了。还是源码看着顺眼。

VScode 继承了 visual stdio 的优点，剔除了冗杂的IDE部分。这些模块都以插件的形式放在插件模块下供你下载。
